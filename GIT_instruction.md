# Инструкция по работе с Git

## Git-команды:

> **git init** - инициализация выбранной папки в качестве пустого репозитория

> **git add filename** - добавление изменений в файл

> **git add .** - добавление изменений сразу во все файлы

> **git commit -m "change description"** - сохранение изменений в файле

> **git status** - узнать текущий статус файла

> **git diff** - разница между последними сделанными изменениями в файле и последней сохраненной версией файла

> **git log** - журнал всех изменений

> **git checkout** - переход к нужной сохраненной версии файла

> **git checkout master** - переход к актуальной версии файла

*Зеленая вертикальная полоса по левому краю* - сигнал, что эта информация еще не закоммичена.

# Работа с ветками:

> **git branch** - выводит список веток в репозитории (текущая ветка выделена *)

> **git branch branch_name** - создание ветки с именем "branch_name"

> **git checkout branch_name** - переход в ветку с именем "branch_name"

> **git merge branch_name** - сливание ветки по именем "branch_name" с текущей, откуда вызывается команда

> **git branch -d branch_name** - удаление ветки с именем "branch_name"

> **git log --graph** - отображение всех коммитов в виде дерева (слева вертикальные линии - это ветки, в которых были сделаны коммиты)

# Экспресс-команды:

> **git log --oneline** - выводит список вcех коммитов в укороченном виде

> **git commit -am "описание коммита"** - объединенная команда **git add "имя файла"** и **git commit -m "описание коммита"**

> **git checkout -b branch_name** - создание новой ветки с именем "branch_name" и сразу же переход в эту новую ветку 

> **git branch -d branch_name** - удаление ветки с именем "branch_name"; *Примечание: эта команда не сработает, если предварительно удаляемая ветка не слилась с основной*  

> **git branch -D branch_name** - *эта команда удалит ветку СРАЗУ, БЕЗ ПРЕДУПРЕЖДЕНИЙ ОБ ОТСУТСТВИИ ПРЕДВАРИТЕЛЬНОГО СЛИЯНИЯ* 

> **git log --graph** - отображение всех коммитов в виде дерева (слева вертикальные линии - это ветки, в которых были сделаны коммиты; *Примечание: вертикальные линии слева будут визуально делиться, если при слиянии веток произошел конфликт*)

# Работа с удаленным репозиторием

> **git clone** - копирование удаленного репозитория (например из Github.com) в свой локальный репозиторий. Эта команда создаст в локальном репозитории новую папку с таким же именем и содержимым, как и на Github.

Чтобы начать работу в скопированной (склонированной) версии репозитория, нужно **не забыть** перейти в эту папку через команду ***cd <имя папки>***

## Передача своего локального репозитория в удаленный (например на github.com) и обратно
1. Необходимо создать/иметь свой аккаунт на Github
2. Создать новый репозиторий (зеленая кнопка **New repository** на Github)
3. Далее Github предложит опции на выбор - что можно сделать, чтобы начать работать с этим созданным репозиторием:

        a. создать новый репозиторий в командной строке(create a new repository on the command line)

        b. связать существующий репозиторий (push an existing repository from the command line)

        c. импортировать код из другого репозитория (import code from another repository)
4. В нашем случае следует прописать рекомендованные команды:
        
> **git remote add origin <ссылка на удаленный репозиторий>** - так мы связываем наш локальный репозиторий с удаленным (дословно мы даем команду Гиту - добавь удаленный репозиторий с именем **origin**, находящийся **<по вот такому адресу>**)

> **git branch -M main** - так указываем, какая ветка будет основной

> **git push -u origin main** - так даем команду Гиту“направить” **(push - вытолкнуть, направить)** все, что создали в своем локальном репозитории, в удаленный с именем **origin** в ветку **main** 

**Примечание!** При первом использовании команды **git push** потребуется авторизация (процесс с подсказками от VSCode) для дальнейшей постоянной связи локального и внешнего репозиториев

***Находясь в своем репозитории на Github, можно внести изменения в свои файлы (справа есть значок “карандаша”) и сохранить эти изменения (зеленая кнопка Commit changes).*** 

> **git pull** - (pull - стянуть) эта команда нужна, чтобы взять актуальную версию удаленного репозитория и синхронизировать с локальной версией. Эта команда является составной - не только подгрузит все изменения с удаленного репозитория, но и попытается слить два состояния двух версий в одну ветку.

При вызове команды **git log** отображаются все коммиты, выполненные в этом репозитории до настоящего момента, с указанием их авторов - это очень удобно отслеживать.

## Как делать Pull request (запрос на вливание изменений в чужой репозиторий):

1. Необходимо перейти по ссылке на репозиторий, в который нужно предложить изменения, и нажать кнопку **Fork**, тем самым создав копию (ответвление) этого репозитория в своем удаленном репозитории.

2. Необходимо склонировать этот репозиторий из своего удаленного в свой локальный репозиторий через команду **git clone**

3. Создать новую ветку и перейти в нее через команду **git checkout -b <branch_name>**

4. Внести в этой ветке предлагаемые изменения

5. Сохранить эти изменения через команды **git add <file_name>** и **git commit -m "change description"**

6. Отправить измененную версия репозитория назад в свой удаленный репозиторий через команду **git push**

**Важно!** Если делать эту команду напрямую из соседней ветки (специально созданной для изменений - см. выше), а не из основной **master/main**, То Гит послкажет команду **git push — set-upstream origin <имя ветки>**

7. Наш удаленный репозиторий обновится версией с изменениями, в том числе появится ветка, в которой были сделаны изменения

8. Можно отправлять запрос на вливание (кнопка **Compare and pull request**). После Github проанализирует предлагаемые изменения и если не обнаружит никаких конфликтов, то высветится сообщение: ***Able to merge. These branches can be automatically merged.***

9. Можно добавить описание предлагаемых изменений в специальном поле и в конце нажать кнопку - **Create pull request**

### Дополнительные команды:

> **git remote -v** - команда показывает все репозитории для записи (fetch) и чтения (push)

> **git remote show origin** - более подробная информация по удаленному репозиторию
