# Инструкция по работе с Git
![Git logo](https://git.codeandcoke.com/lib/exe/fetch.php?cache=&media=git-github.png)
## Установка Git
##### Unix-подобных ОС (Ubuntu, Debian):
#
```
sudo apt-get update
sudo apt-get install git
```
##### Windows:
Загрузите установщик с [официального сайта Git](https://git-scm.com) и выполните установку.
#
#
#
## Настройка Git
После установки Git укажите ваше имя и email:

```
git config --global user.name "Ваше Имя"
git config --global user.email "ваша_почта@example.com"
```

## Работа с репозиториием
Создание нового репозитория
```
git init
```
Клонирование существующего репозитория
```
git clone https://github.com/username/repo-name.git
```
Добавление файлов в репозиторий
```
git add <имя_файла>
git add .
```
Создание коммитов
```
git commit -m "Сообщение коммита"
```
Просмотр состояния репозитория
```
git status
```
Просмотр истории коммитов
```
git log
```
#
#
#
## Взаимодействие с удалённым репозиторием
Отправка изменений
```
git push origin main
```
Получение изменений
```
git pull
```
#
#
#
## Работа с ветками
Создание ветки
```
git branch <имя_ветки>
```
Переключение между ветками
```
git checkout <имя_ветки>
```
Слияние веток
```
git merge <имя_ветки>
```
Удаление ветки
```
git branch -d <имя_ветки>
```
Удаление репозитория
```
git remote rm origin
```
#
#
#
## Откат изменений
Отмена последнего коммита (с сохранением изменений в файлах)
```
git reset --soft HEAD^
```
Полный откат последнего коммита (с удалением изменений)
```
git reset --hard HEAD^
```
Откат определенного файла до состояния последнего коммита
```
git checkout HEAD <имя_файла>
```
#
#
#
## Работа с .gitignore
Чтобы игнорировать определенные файлы или папки в вашем репозитории, создайте файл .gitignore и добавьте туда пути к игнорируемым объектам:
```
Игнорировать все .txt файлы
*.txt

Не игнорировать конкретный файл
!example.txt

Игнорировать папку logs
logs/
```


## Разрешение конфликтов
Конфликты слияния возникают, когда одни и те же части файла были изменены в двух разных ветках. Git попросит вас вручную разрешить эти конфликты. После разрешения необходимо сделать коммит, чтобы завершить слияние.
## Советы и хитрости
Просмотр изменений перед коммитом:
```
git diff
```
Использование алиасов для упрощения команд Git:
```
git config --global alias.st status
```
Хранение часто используемых команд в .gitconfig поможет экономить время.

some text for master branch
some text to secondary branch
