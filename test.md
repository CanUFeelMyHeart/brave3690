## Команды
> **```git init```** - создание пустого репозитория в выбранной папке.

> **```git add .```** - сохранить все файлы , добавленные в данный репозиторий. Прим. 30 картинок.

> Зелёный цвет сигнализирует о том, что информация не сохранена.

> **```git commit -am```** - сочетание комманд *git add* и *git commit -m*, где *-m* - флаг message или указаний изменения.

> **```git add```** - добавляет текущее содержимое в файл.  

> **```git status```** - получить информацию от git о его текущем состоянии.

> **```git commit -m “message”```** – запись изменений в репозиторий.

> **```git log```** – вывод на экран истории всех коммитов (изменений) с их хеш-кодами.

> **```git checkout```** – переход к коммиту по хешу (от 4-х первых символов хеша)

> **```git checkout master```** – вернуться к актуальному состоянию.

> **```git diff```** – увидеть разницу между текущим файлом и закоммиченным файлом.

> **```git log --oneline```** – вывести изменения списком.

> **```git log --oneline --graph```** – вывести изменения списком с графом изменений веток.

> **```cd <adress>```** – (change directory) позволяет перейти по указанному адресу в терминале (Linux/Mac).

> **```cd ..```** – (change directory) позволяет вернуться в предыдущую директорию в терминале (Linux/Mac).

> **```sh <code>```** – позволяет записать программный код в специальный блок с подсветкой, что делает его более читабельным.
sh необходима для подсветки кода.
Пример:

```sh
git commit -m “message”
```

> **```git clone <address>```** – Скопировать репозиторий в локальную папку.

> **```mkdir <folder_name>```** – создать новую папку в выбранной директории через терминал (Linux/Mac).

**```git branch -M main```** – переименовать ветку master в main.

**```git remote add origin https://address.git```** – связать локальны репозиторий со своим GitHub.

**```git remote show```** – задан-ли репозиторий.

**```git remote show origin```** (подробно) или **```git remote -v```** – отобразить адресс репозитория для извлечения и отправки.

**```git push -u origin main```** – (push - толкать) отправить изменения ветки main в репозиторий (origin).

**```git pull```** – (pull - тянуть) выгрузить изменения с сервера в локальный репозиторий.

**```git push```** – (push - толкать) выгрузить изменения с локального репозитория на сервер.

**```git branch -d <branch_name>```** – удаление ветки в локальном репозитории.

**```git push origin --delete <branch_name>```** – удаление ветки в репозитории GitHub.

**```git pull --rebase```** – при конфликте, когда изменения есть и в GitHub, и локально, выкачивает данные с сервера.
 
**fork (вилка)** – возможность GitHub продублировать репозиторий другого автора у себя и предложить изменение в дальнейшем.
