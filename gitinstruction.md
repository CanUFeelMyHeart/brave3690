# Инструкция для работы с Git 
Git - система хранения и ведения версий.
## Создание репозитория
Для создания репозитория используется команда *git init* при этом в папке репозитория появится скрытая папка .git
Теперь в репозитории можно создать или добавить файлы, версии которых необходимо сохранять и контролировать.
## Создание коммитов

Примечание. После изменения в файле , если Вы работаете в редакторе "VS Code" выполните сохранение __CTRL + S__.

1. Выполнить команду *git add <имя файла>* - добавление изменений (файл фудет проиндексирован).

2. Выполнить команду *git commit -m "ваш комментарий" - создание коммита (фиксации), с англ. - "совершать, фиксировать".

## Просмотр состояния репозитория
Для просмотра состояния репозитория используется команда *git status*. Команда позволяет узнать, были изменения в файлах.

## Просмотр всех действий в репозитории
Для просмотра всех действий в созданном нами репозитории используется команда *git log*. При этом мы увидим полный идентификатор - хэш, состоящий из 40 символов.
Для отображения укороченного варианта хэша используется команда *git log --oneline*.

## Перейти на какой либо коммит (зафиксированное состояние)
Для перехода на какой либо коммит, используется команда *git checkout 7изначныйхэш*

## Переключиться на самое финальное состояние
Для переключения используется команда *git checkout master*

## Показать отличия при изменении и сохранении
Для отображения отличий используется команда *git diff*

## Отменить изменения до уровня последнего коммита
Для отмены изменений до состояния последнего коммита используется команда *git restore*

##  ВЕТКИ 

### Создание ветки
Для создания новой ветки сохранений (коммитов) используется команда 
```sh
git branch имяветки
```

### Показать все ветки
Для того чтобы посмотреть все ветки используется команда 
```sh
git branch
```
, будет показан список веток и звёздочкой отмечена текущая ветка.

### Переключиться на ветку
Используется команда
```sh
git checkout имя_ветки
```
, эта же команда использовалась ранее для переключения в состояние последнего коммита.

### Слияние веток
Используется команда 
```sh
git merge имяветки
```

### Удаление ветки
Используется команда
```sh
git branch -d имяветки
``` 

### Визуализировать ветки
Используйте команду
```sh
git log --graph
```
 

### Сочетанае сокращённого лога и визуализация ветки
Используйте команду
```sh
git log --oneline --graph
```

## Работа с удалённым репозиторием

### Команда клонировать удалённый репозиторий на ресурсе Github в локальный Git

```
git clone "путь до репозитория"

```

### Переименование ветки локального репозитория
```
git branch -M main
```
В локальном репозитории git главная ветка master
В удалённом репозитории github главная ветка main

### Подключение локального репозитория к удалённому
```
git remote add origin "имя репозитория"
```

### Покажет какой удалённый репозиторий задан.

```
git remote show
```
более детально 

```
ключ -v 
```

### Вытолкнуть на удалённый сервер github изменения в локальном репозитории

```
git push
```

### Вытолкнуть  все изменения в локальном репозитори, в репозиторий на сервере github, в ветку main

```
git push -u origin main
```

### Показывает более подробную иформацию о удалённом репозитории
```
git remote show origin
```

### Выталкивает определённую ветку
```
git push --set -upstream origin "имя ветки"
```

### Удалить ветвь удалённого репозитория
```
git push origin --delite "имя ветки"
```

## Работа с репозиториями в случае конфликтов

### 1. выталкиваем локальный в удалённый
```
git push
```
получаем отказ!

### 2. команда забирает изменения удалённого репозитория в локальный и предлагает решить конфликт в редакторе слияния
```
git pull --rebase
```
далее решаем отказ в редакторе слияния

### 3. Фиксируем изменения
```
git add "файл"
git commit -m "коментарий"
```

### 4. Продолжаем
```
git rebase --continue
```

### 5. И опять выталкиваем (без конфликтов)
```
git push
```

## Копирование чужого репозитория в Github с помощью команды Fork, клонируем эту копию в локальный, внесение изменений и выталквание обратно, после чего pull request запрос держателю оригинала на внесение изменений.

 1. Делаем fork репозитория, в которой потом хотим сделать pull request. Ищем кнопку Fork на странице репозитория.

 2. Выполняем команду клонирования из своей fork-копии в локальный репозиторий
 ```
 git clone "путь до репозитория"
 ```
 3. Создаем новую ветку и вносим необходимые изменения в файл
 ```
 git branch "имя новой ветки"
 git checkout "имя новой ветки"
 вносим изменения в файле
 git add README.md
 git commit -m "комментарий - фиксируем изменение"
 ```
 4. Выталкиваем (с или без конфликтов)
 ```
 git push --set-upstream origin "имя новой ветки"
 git pull --rebase
 git add "имя файла"
 git commit -m "комментарий"
 git rebase --continue
 git push
 ```
 5. Переходим на свою страницу репозитория. Выбираем новую ветку "имя новой ветки" и жмем кнопку __Compare & pull request__.

Также pull request можно сделать нажа на кнопку __Contribute__ далее __Open pull request__.
